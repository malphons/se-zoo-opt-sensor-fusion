Sensor Fusion

E06 Optimal Filtering Sensor Fusion 2-Link Arm Est. 1970

Fusing complementary sensors yields better estimates than any sensor alone. Each sensor adds positive-definite information, so fusion can only reduce uncertainty.

Prerequisite: E05: Extended Kalman Filter

Overview

Real robots carry multiple sensors, each with different strengths and weaknesses. An encoder precisely measures joint position but tells you nothing about velocity. A gyroscope directly measures angular rate but drifts over time. An accelerometer captures fast dynamics but is plagued by noise and gravity coupling. No single sensor gives you the full picture.

Sensor fusion combines these complementary measurements into a single estimate that is better than any individual sensor. The key mathematical insight comes from the information form of the Kalman filter: each sensor contributes a positive-definite information matrix $\mathbf{I}_k = \mathbf{H}^T \mathbf{R}^{-1} \mathbf{H}$, and these simply add.

Key Insight

Because each sensor's information matrix $\mathbf{H}^T \mathbf{R}^{-1} \mathbf{H}$ is positive semi-definite, adding more sensors can only increase total information and decrease uncertainty. Fusion never makes things worse — even a noisy sensor helps.

Key Properties

  • Complementary sensing — Different sensors observe different state components
  • Precision weighting — Low-noise sensors automatically get more weight
  • Information additivity — Information matrices add directly; fusion is a sum
  • Guaranteed improvement — Fused covariance is always smaller than any individual sensor's covariance
  • Graceful degradation — If one sensor fails, the others still provide partial information

Sensors in This Demo

  • Encoder (green) — Measures position $\theta$, low noise ($\sigma \approx 0.02$ rad)
  • Gyroscope (purple) — Measures velocity $\dot\theta$, moderate noise ($\sigma \approx 0.08$ rad/s)
  • Accelerometer (yellow) — Measures acceleration $\ddot\theta$, high noise ($\sigma \approx 0.30$ rad/s²)

How It Works

  1. Predict: Propagate the state estimate through the nonlinear pendulum dynamics and grow the covariance by adding process noise $\mathbf{Q}$.
  2. Convert to information form: Compute the predicted information matrix $\mathbf{I}^{-} = (\mathbf{P}^{-})^{-1}$ and information vector $\mathbf{i}^{-} = \mathbf{I}^{-}\hat{\mathbf{x}}^{-}$.
  3. Compute each sensor's contribution: For sensor $j$, form $\mathbf{I}_j = \mathbf{H}_j^T \mathbf{R}_j^{-1} \mathbf{H}_j$ and $\mathbf{i}_j = \mathbf{H}_j^T \mathbf{R}_j^{-1} z_j$.
  4. Fuse by addition: $\mathbf{I}^{+} = \mathbf{I}^{-} + \sum_j \mathbf{I}_j$ and $\mathbf{i}^{+} = \mathbf{i}^{-} + \sum_j \mathbf{i}_j$.
  5. Recover estimates: $\mathbf{P}^{+} = (\mathbf{I}^{+})^{-1}$ and $\hat{\mathbf{x}}^{+} = \mathbf{P}^{+}\mathbf{i}^{+}$.

The beauty of the information form is that step 4 is just matrix addition — no matrix inversion is needed until the final recovery step. Each sensor's contribution is independent, making it trivial to add or remove sensors.

Why Information Form?

In the standard covariance form, fusing $N$ sensors requires $N$ sequential Kalman updates. In the information form, you simply add all information matrices and vectors, then invert once. This is mathematically equivalent but conceptually cleaner and naturally parallelizable.

Key Equations

Information Matrix from Sensor $j$

$$\mathbf{I}_j = \mathbf{H}_j^T \mathbf{R}_j^{-1} \mathbf{H}_j$$

Each sensor contributes a positive semi-definite information matrix. $\mathbf{H}_j$ is the observation matrix and $\mathbf{R}_j$ is the measurement noise covariance.

Information Vector from Sensor $j$

$$\mathbf{i}_j = \mathbf{H}_j^T \mathbf{R}_j^{-1} \mathbf{z}_j$$

Fusion by Addition

$$\mathbf{I}_{\text{fused}} = \mathbf{I}_{\text{prior}} + \sum_{j=1}^{N_s} \mathbf{I}_j \qquad \mathbf{i}_{\text{fused}} = \mathbf{i}_{\text{prior}} + \sum_{j=1}^{N_s} \mathbf{i}_j$$

Recovery of State and Covariance

$$\mathbf{P}_{\text{fused}} = \mathbf{I}_{\text{fused}}^{-1} \qquad \hat{\mathbf{x}}_{\text{fused}} = \mathbf{P}_{\text{fused}} \, \mathbf{i}_{\text{fused}}$$

Sensor Observation Models (This Demo)

$$\text{Encoder: } \mathbf{H}_{\text{enc}} = \begin{bmatrix}1 & 0\end{bmatrix}, \quad R_{\text{enc}} = \sigma_{\text{enc}}^2$$

$$\text{Gyro: } \mathbf{H}_{\text{gyro}} = \begin{bmatrix}0 & 1\end{bmatrix}, \quad R_{\text{gyro}} = \sigma_{\text{gyro}}^2$$

$$\text{Accel: } \mathbf{H}_{\text{acc}} = \begin{bmatrix}-\tfrac{g}{L}\cos\theta & -d\end{bmatrix}, \quad R_{\text{acc}} = \sigma_{\text{acc}}^2$$

Positive-Definiteness Guarantee

$$\mathbf{I}_j = \mathbf{H}_j^T \mathbf{R}_j^{-1} \mathbf{H}_j \succeq 0 \quad \Rightarrow \quad \mathbf{I}_{\text{fused}} \succeq \mathbf{I}_{\text{prior}} \quad \Rightarrow \quad \mathbf{P}_{\text{fused}} \preceq \mathbf{P}_{\text{prior}}$$

Adding any sensor can only decrease (or maintain) the fused covariance. More sensors always help.

MATLAB Implementation

MATLAB
%% E06: Sensor Fusion — Information Form (3 Sensors)
% Pendulum: state = [theta; omega], 3 complementary sensors

dt = 0.02; N = 250; g = 9.81; L = 1.0; d = 0.15;

% Sensor noise standard deviations
sig_enc = 0.02; sig_gyro = 0.08; sig_acc = 0.30;
R_enc = sig_enc^2; R_gyro = sig_gyro^2; R_acc = sig_acc^2;

% Process noise
Q = diag([0.0025, 0.01]);

% Initial conditions
x = [0.8; 0]; xhat = [0; 0]; P = eye(2);

for k = 1:N
    % --- Measurements ---
    z_enc  = x(1) + sig_enc*randn;
    z_gyro = x(2) + sig_gyro*randn;
    a_true = -(g/L)*sin(x(1)) - d*x(2);
    z_acc  = a_true + sig_acc*randn;

    % --- Predict ---
    alpha = -(g/L)*sin(xhat(1)) - d*xhat(2);
    xhat_m = [xhat(1)+xhat(2)*dt; xhat(2)+alpha*dt];
    F = [1, dt; -(g/L)*cos(xhat(1))*dt, 1-d*dt];
    P_m = F*P*F' + Q;

    % --- Information form fusion ---
    I_prior = inv(P_m);
    i_prior = I_prior * xhat_m;

    % Encoder: H = [1 0]
    I_enc = [1/R_enc, 0; 0, 0];
    i_enc = [z_enc/R_enc; 0];

    % Gyro: H = [0 1]
    I_gyro = [0, 0; 0, 1/R_gyro];
    i_gyro = [0; z_gyro/R_gyro];

    % Accelerometer: H_acc = [-(g/L)*cos(th), -d]
    cth = cos(xhat_m(1));
    H_acc = [-(g/L)*cth, -d];
    I_acc = H_acc'/R_acc * H_acc;
    a_pred = H_acc * xhat_m;
    a_nl = -(g/L)*sin(xhat_m(1)) - d*xhat_m(2);
    z_eff = z_acc - a_nl + a_pred;
    i_acc = H_acc'/R_acc * z_eff;

    % Fuse: sum all information
    I_fused = I_prior + I_enc + I_gyro + I_acc;
    i_fused = i_prior + i_enc + i_gyro + i_acc;

    % Recover state and covariance
    P = inv(I_fused);
    xhat = P * i_fused;

    % --- True dynamics + noise ---
    a_x = -(g/L)*sin(x(1)) - d*x(2);
    x = [x(1)+x(2)*dt; x(2)+a_x*dt] + chol(Q)'*randn(2,1);
end
